# 논문용 통합 결과표

- 분석대상: 생성형 AI 사용 경험자 377명
- 연구모형: motivation, support_main(Q16_1~Q16_4 평균) -> effect -> Q20_1~Q20_3
- 통제변수: gender, rank_code, career_code

종속변수는 분석 목적에 따라 두 층으로 봐야 합니다.

엄밀히 말하면 메인 연구모형의 최종 종속변수는 3개입니다.

- Q20_1: 업무효율 개선 기대

- Q20_3: 반복업무 자동화 기대

여기에 Q20_4는 추가 분석용 보조 종속변수입니다.

- Q20_4: 일자리 대체 인식
  즉, “최종 결과변수”는 메인 3개 + 보조 1개로 총 4개라고 볼 수 있지만, 논문 메인 가설 검증용 종속변수는 3개라고 정리하
  는 게 더 정확합니다.

변수 간 관계를 다시 정리하면 이렇습니다.

1. 독립변수

- motivation: 자발적 AI 활용동기
- support_main: 조직지원 인식

2. 매개변수

- effect: 인식된 업무효과

3. 종속변수

- 메인 종속변수: Q20_1, Q20_2, Q20_3
- 보조 종속변수: Q20_4

- motivation -> effect
- support_main -> effect
- motivation -> Q20_1, Q20_2, Q20_3
- support_main -> Q20_1, Q20_2, Q20_3
- effect -> Q20_1, Q20_2, Q20_3

즉, motivation과 support_main이 병렬 독립변수이고, effect가 공통 매개변수이며, 최종적으로 Q20_1~Q20_3을 설명하는 구조
입니다.

Q20_4는 같은 방식으로 추가로 돌려본 보조 결과변수입니다.

- motivation -> Q20_4
- support_main -> Q20_4
- effect -> Q20_4

## 위계적 회귀분석

표 6~9는 같은 방식의 회귀를 종속변수만 바꿔서 단계적으로 보여주는 표입니다. 핵심은 “통제변수만 넣었을 때”와 “주요 독립
변수를 추가했을 때”, 그리고 “매개변수(effect)까지 넣었을 때” 설명력이 얼마나 늘고 계수가 어떻게 바뀌는지를 보는 것입니
다.

표 6은 종속변수가 effect입니다. 즉, 자발적 AI 활용동기(motivation)와 조직지원 인식(support_main)이 인식된 업무효과를
얼마나 설명하는지 보는 표입니다. 여기서 motivation, support_main이 유의하면 두 변수가 매개변수인 effect를 형성하는 선
행요인이라는 뜻입니다.

표 7은 종속변수가 Q20_1이고, “AI가 업무효율을 개선할 것이라는 기대”를 설명하는 모형입니다.

- 모형 1: 통제변수만 투입
  즉, 업무효율 개선 기대가 개인 동기와 조직지원에 의해 직접 설명되는지, 아니면 업무효과 인식을 거쳐 설명되는지를 보는
  표입니다.

표 8은 종속변수가 Q20_2이고, “AI가 의사결정을 지원할 것이라는 기대”를 설명하는 모형입니다. 구조는 표 7과 동일합니다.
여기서는 특히 effect를 넣었을 때 support_main의 직접효과가 약해지는지, motivation의 영향이 유지되는지를 보면 매개 패턴
을 해석할 수 있습니다.

표 9는 종속변수가 Q20_3이고, “AI가 반복업무를 자동화할 것이라는 기대”를 설명하는 모형입니다. 역시 같은 3단계 구조이고,
이 표는 자동화 기대가 motivation보다 support_main에 더 크게 좌우되는지, 또 effect가 추가되었을 때 설명력이 얼마나 증가
하는지를 보여줍니다.

정리하면,

- 표 6: 독립변수들이 매개변수 effect를 설명하는 표

- 표 8: 의사결정 지원 기대(Q20_2)의 위계적 회귀

즉, **매개변수는 어떤 식에서는 종속변수이고, 다른 식에서는 독립변수(예측변수)**가 됩니다.

- 매개모형에서는 effect가 종속변수이므로 VIF 계산 대상 아님
- 직접효과모형에서는 effect가 예측변수로 들어가므로 VIF 계산 대상 맞음

따라서 표 10에서 effect의 VIF가 나온 것은, Q20_1~Q20_3 회귀식에서는 effect가 종속변수가 아니라 독립변수 쪽에 포함되기
때문입니다.

논문식으로 짧게 쓰면:
“다중공선성은 각 회귀모형에 동시에 투입된 예측변수들에 대해 점검하였으며, 종속변수는 진단 대상에서 제외하였다. 매개변
수인 effect는 매개모형에서는 종속변수로, 직접효과모형에서는 예측변수로 기능하므로 후자의 경우에 한해 공선성 진단에 포
함하였다.”

direct_motivation, indirect_motivation, total_motivation은 motivation이 각 종속변수에 미치는 효과를 직접효과,
간접효과, 총효과로 나눈 것

기준 구조는 이겁니다.

- motivation -> effect -> Q20
- 동시에 motivation -> Q20

그래서 의미는 다음과 같습니다.

direct_motivation

- effect를 통제한 뒤에도 motivation이 종속변수에 직접 미치는 효과입니다.
- 즉, 매개변수 effect를 거치지 않는 경로입니다.
- 회귀식으로는 Q20 ~ motivation + support_main + effect + controls에서의 motivation 계수입니다.

indirect_motivation

- motivation이 effect를 거쳐 종속변수에 미치는 간접효과입니다.
- 즉, motivation -> effect 경로와 effect -> Q20 경로를 곱한 값입니다.
- “동기가 높을수록 업무효과를 더 크게 느끼고, 그 업무효과 인식이 다시 기대를 높인다”는 경로입니다.

total_motivation

- motivation이 종속변수에 미치는 전체 효과입니다.
- 직접효과 + 간접효과입니다.
- 회귀식으로는 Q20 ~ motivation + support_main + controls에서의 motivation 계수와 대응합니다.

예를 들어 Q20_1에서 보면

- indirect_motivation = 0.157
- total_motivation = 0.523

뜻은,

- 동기는 업무효율 개선 기대에 직접적으로도 정(+)의 영향을 미치고
- 업무효과 인식을 통해서도 추가적인 정(+)의 영향을 미치며
- 둘을 합친 전체 영향은 0.523이라는 뜻입니다.

같은 방식으로

- direct_support_main
- indirect_support_main
- total_support_main
  도 support_main의 직접효과, 간접효과, 총효과를 뜻합니다.

짧게 정리하면:

- direct = 매개변수를 거치지 않는 효과
- indirect = 매개변수를 거치는 효과
- total = direct + indirect
